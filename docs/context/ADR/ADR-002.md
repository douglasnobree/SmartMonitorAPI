# ADR-002

Status: Accepted
Data: 2026-05-21

## Contexto
A API requer autenticacao para endpoints de negocio e integra com backend principal. [Fonte: README][Fonte: codigo]

## Decisao
Usar JWT (SimpleJWT) e aceitar token no header Authorization sem prefixo Bearer. [Fonte: codigo]

## Alternativas consideradas
- JWT padrao com prefixo Bearer. [Inferencia]
- Session auth. [Inferencia]

## Consequencias positivas
- Simplicidade para clientes que enviam token bruto. [Fonte: codigo]

## Consequencias negativas
- Divergencia de padrao HTTP Bearer pode confundir integracoes. [Inferencia]

## Impacto tecnico
- CustomJWTAuthentication substitui get_header e get_raw_token. [Fonte: codigo]

## Quando revisar
Se houver integracoes externas que exigem Bearer ou padrao OAuth. [Inferencia]
