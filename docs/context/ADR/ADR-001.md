# ADR-001

Status: Accepted
Data: 2026-05-21

## Contexto
A API necessita expor endpoints REST com autenticacao e documentacao. [Fonte: README][Fonte: codigo]

## Decisao
Usar Django + Django REST Framework como base da API. [Fonte: README][Fonte: codigo]

## Alternativas consideradas
- Flask/FastAPI para API leve. [Inferencia]
- Node/Express. [Inferencia]

## Consequencias positivas
- Ecosistema maduro com admin e autenticacao prontos. [Inferencia]
- Integracao direta com drf-yasg para Swagger/Redoc. [Fonte: codigo]

## Consequencias negativas
- Overhead de framework para servicos simples. [Inferencia]

## Impacto tecnico
- Estrutura de apps Django e configuracoes em projectSM. [Fonte: codigo]

## Quando revisar
Quando houver necessidade de alta performance ou reescrita da API. [Inferencia]
